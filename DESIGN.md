# Toybox Trio / 玩具箱三重奏 — Game Design Document

版本：0.3 三车教学与三消反馈原型
目标平台：桌面与移动浏览器，优先 Poki  
技术栈：Three.js + TypeScript + Vite

## 1. 产品概念

玩家面对一个塞满彩色玩具的 3D 玩具箱，通过点击收集玩具。收纳槽中出现三个相同玩具时，它们会自动归位消除。清空玩具箱后，救出压在箱底的方块宠物。

本作借鉴“3D 杂物堆 + 三同物收集”的广义玩法，但采用原创名称、玩具收纳主题、宠物救援目标、视觉系统和关卡节奏，不使用《抓大鹅》的名称、火锅包装、鹅形象、界面或资产。

## 2. 设计支柱

1. **三秒理解**：点击玩具；三个相同玩具自动归位。
2. **触觉式 3D 找物**：拖动旋转玩具堆，观察遮挡和轮廓。
3. **紧张但公平**：七格收纳槽制造压力；牌库数量始终可被三整除。
4. **可见的长期目标**：每箱救出一只宠物，形成连续关卡动力。
5. **网页优先**：低包体、短加载、单指操作、响应式 HUD；模型、图片、字体和音频均本地化，仅加载 Poki 官方 SDK。

## 3. 核心循环

1. 进入第一箱，看到三辆玩具车；长时间没有选择时才出现点击提示。
2. 点击玩具，3D 玩具从物理堆飞入七格 DOM 收纳槽，再变成清晰的 2D 图标。
3. 凑齐三个相同玩具，三个槽位图标向中心聚拢并爆开，获得分数并释放槽位。
4. 拖动旋转玩具堆，寻找被遮挡的目标。
5. 必要时使用“抖一抖”重新排列未收集玩具，或撤回最后一个选择。
6. 收纳槽满且没有形成三消则失败；清空全部玩具则成功。
7. 宠物登场庆祝，进入下一箱。

## 4. 首局与三分钟目标

第一箱是独立教学关，仅包含 3 辆相同玩具车。模型使用正常游戏材质，不再使用发光、脉冲或黑色描边；玩家可以直接点击任意一辆车。可操作后 5 秒仍未选择，界面才短暂提示“点击车模型，把它放进下方收纳槽”。完成唯一一次三消后立即通关并进入正式关卡。

- 0–10 秒：加载完成，点击“开始整理”。
- 10–25 秒：选择三辆车，观察模型飞入槽位并自动消除。
- 25–90 秒：进入第二箱，理解七格槽位和旋转操作。
- 90–210 秒：处理遮挡和种类混合，使用一次工具。
- 180–300 秒：完成第一箱或失败后快速重试。

Poki Player Fit 的正式过线指标是平均游玩超过 3 分钟，且至少 25% 的玩家超过 3 分钟。内部目标更保守：平均 4 分钟以上，35% 以上玩家超过 3 分钟。

## 7. 操作与相机

- 点击或轻触：选择射线命中的最前方玩具。
- 水平拖动：相机绕玩具托盘的 Y 轴旋转，物理世界保持稳定。
- Escape / 暂停按钮：暂停或继续。
- 抖一抖：向仍在托盘中的刚体施加短促冲量，不改变牌库。
- 撤回：将收纳槽最后一件玩具送回堆顶。

相机固定为俯视透视视角。竖屏时自动后退并提高，确保玩具箱和底部收纳槽同时可见。

每次点击开始或重新开始都会生成新的运行种子；牌库顺序、初始朝向和物理落点随之改变。关卡数量和每种玩具的三倍数约束保持不变；第一箱固定为三辆可见的车，因此随机变化不会破坏可解性或首局教学。

## 8. UI 与视觉方向

视觉语言是“柔软塑料玩具 + 半透明收纳标签”：紫色玩具箱、糖果色玩具、圆角磨砂 HUD。3D 画布保留中央区域，HUD 只占顶部边缘、右侧工具轨和底部关键收纳槽。

收集与消除采用 3D/2D 混合表现：玩家先看到真实模型脱离物理堆并飞向对应屏幕槽位，抵达后才切换成稳定易读的 DOM 图标；第三件到达时三个图标向共同中心收拢、缩小，并配合彩色粒子爆开。这既保留“拿起实体”的空间感，也让三消结果在手机屏幕上足够清楚。

游戏根据浏览器语言自动选择英语、法语、意大利语、德语、西班牙语、简体中文、日语、韩语、巴西葡语、俄语或土耳其语；无法匹配时回退英语。全部文案集中在 `i18n.ts`，字体使用系统字体，避免 Poki 对外部字体请求的限制。

## 9. 音频

全部音效由 Web Audio API 即时合成，不下载音频文件：

- 选择：短促正弦音。
- 三消：随连击升高的三音和弦。
- 抖动、撤回、胜利和失败：独立合成反馈。

首次点击开始按钮时解锁 AudioContext，并提供静音按钮。

## 10. 技术架构

- `simulation/GameState.ts`：牌库、收纳槽、匹配、胜负与分数，完全不依赖 Three.js。
- `ToyboxGame.ts`：输入、场景适配、关卡流程和动画编排。
- `physics/PilePhysics.ts`：Rapier 世界、托盘边界、玩具碰撞体、固定步长同步和物理摇晃。
- `render/ToyFactory.ts`：GLB 加载、归一化、克隆和程序化模型。
- `render/TweenSystem.ts`：轻量、帧同步的补间动画。
- `i18n.ts`：Poki 推荐语言文案、浏览器语言解析和英语回退。
- `ui/GameUI.ts`：DOM HUD、响应式布局、提示和结果界面。
- `platform/PokiBridge.ts`：Poki 初始化、生命周期与商业插播桥接；本地无 SDK 时安全降级。

物件先按每局随机种子生成分层位置和朝向，再由 Rapier 重力、摩擦和碰撞自然落入托盘并形成遮挡关系。第一箱使用居中的三车初始布局，后续关卡恢复分层堆叠。游戏规则仍独立于物理世界；点击继续由 Three.js Raycaster 处理。移动端限制像素比，并允许刚体休眠以控制帧开销。

## 12. Poki 接入

`index.html` 已加载 Poki HTML5 SDK v2，并在游戏资产加载前初始化。资产完成后发送 `gameLoadingFinished()`；开始关卡、继续、暂停、胜负分别发送匹配的 `gameplayStart()` / `gameplayStop()`。开始、重试和下一关会先请求 `commercialBreak()`，广告开始时暂停音频和游戏输入；SDK 缺失、被广告拦截器阻止或 Promise 拒绝时安全继续游戏。

后续可加入：

- `measure()`：关卡开始、首个三消、工具使用、失败和通关。
- 失败后的可选激励广告，例如一次清空两个槽位；必须同时提供普通重开。

## 13. 可玩原型验收标准

- npm 安装、开发运行和生产构建均成功。
- 桌面鼠标及移动触控均能选取和旋转。
- 三件相同玩具必定消除，七格满时失败，清空时胜利。
- 第一箱只有 3 辆正常材质的车；5 秒未选择才显示点击提示，首次选择后提示不再出现。
- 3D 玩具抵达槽位后才出现 2D 图标，三消聚拢和爆开反馈清晰完整。
- 每次开始或重开都会得到不同堆叠，同时保持牌库可三消。
- 抖动和撤回次数正确，失败后可立即重试。
- 四个开场关卡连续可玩，宠物模型正常登场。
- 模型加载失败时仍可使用程序化回退完成游戏。
- 竖屏和横屏均不遮挡关键玩法区域。
- 构建包不发起字体、模型或音频的第三方网络请求。
- Poki 推荐的 11 种语言可切换，未知语言稳定回退英语且关键 HUD 不溢出。
