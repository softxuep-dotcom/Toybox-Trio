# Toybox Trio / 玩具箱三重奏 — Game Design Document

版本：0.2 引导与三消反馈原型
目标平台：桌面与移动浏览器，优先 Poki  
技术栈：Three.js + TypeScript + Vite

## 1. 产品概念

玩家面对一个塞满彩色玩具的 3D 玩具箱，通过点击收集玩具。收纳槽中出现三个相同玩具时，它们会自动归位消除。清空玩具箱后，救出压在箱底的方块宠物。

本作借鉴“3D 杂物堆 + 三同物收集”的广义玩法，但采用原创名称、玩具收纳主题、宠物救援目标、视觉系统和关卡节奏，不使用《抓大鹅》的名称、火锅包装、鹅形象、界面或资产。

## 2. 设计支柱

1. **三秒理解**：点击玩具；三个相同玩具自动归位。
2. **触觉式 3D 找物**：拖动旋转玩具堆，观察遮挡和轮廓。
3. **紧张但公平**：七格收纳槽制造压力；牌库数量始终可被三整除。
4. **可见的长期目标**：每箱救出一只宠物，形成连续关卡动力。
5. **网页优先**：低包体、短加载、单指操作、响应式 HUD、无外部运行时请求。

## 3. 核心循环

1. 进入第一箱，先看到三件带暖色自发光和黑色描边的相同玩具。
2. 点击玩具，3D 玩具从物理堆飞入七格 DOM 收纳槽，再变成清晰的 2D 图标。
3. 凑齐三个相同玩具，三个槽位图标向中心聚拢并爆开，获得分数并释放槽位。
4. 拖动旋转玩具堆，寻找被遮挡的目标。
5. 必要时使用“抖一抖”重新排列未收集玩具，或撤回最后一个选择。
6. 收纳槽满且没有形成三消则失败；清空全部玩具则成功。
7. 宠物登场庆祝，进入下一箱。

## 4. 首局与三分钟目标

首局包含 6 种玩具，每种 6 件，共 36 件、12 组三消。前两组三件组合使用暖黄色自发光、脉冲和黑色反壳描边；教学期间只接受高亮目标。完成第一组后，第二组三件目标会落到堆顶；完成两组后取消限制，让玩家自由整理。

- 0–10 秒：加载完成，点击“开始整理”。
- 10–35 秒：跟随高亮完成第一组三消，观察玩具飞入槽位。
- 35–90 秒：完成第二组，理解自动消除、七格槽位和旋转操作。
- 90–210 秒：处理遮挡和种类混合，使用一次工具。
- 180–300 秒：完成第一箱或失败后快速重试。

Poki Player Fit 的正式过线指标是平均游玩超过 3 分钟，且至少 25% 的玩家超过 3 分钟。内部目标更保守：平均 4 分钟以上，35% 以上玩家超过 3 分钟。

## 5. 关卡结构

| 关卡 | 玩具种类 | 总件数 | 救援宠物 | 工具 |
| --- | ---: | ---: | --- | --- |
| Box 1 | 6 | 36 | Mochi 猫 | 抖动 ×2、撤回 ×1 |
| Box 2 | 7 | 42 | Pip 兔 | 抖动 ×1、撤回 ×1 |
| Box 3 | 9 | 54 | Bao 熊猫 | 抖动 ×1、撤回 ×1 |

第三关后循环宠物，同时保留完整 9 种玩具。后续正式版本应加入不同容器形状、房间主题和新的程序化物件，而不是只增加数量。

## 6. 玩具集合

- 弹力球：球体和装饰环。
- 玩具赛车：Kenney Toy Car Kit GLB。
- 拼搭积木：Kenney Brick Kit GLB。
- 口袋火箭：圆柱、圆锥和尾翼。
- 陀螺：圆锥、圆环和手柄。
- 礼物盒：长方体、缎带和蝴蝶结。
- 小机器人：长方体、胶囊体和球体。
- 玩具鼓：圆柱和圆环。
- 发光星星：挤压星形几何体。

所有匹配物件都有独立轮廓、主色和收纳槽图标。模型加载失败时，赛车、积木和宠物均有程序化几何体回退。

## 7. 操作与相机

- 点击或轻触：选择射线命中的最前方玩具。
- 水平拖动：相机绕玩具托盘的 Y 轴旋转，物理世界保持稳定。
- Escape / 暂停按钮：暂停或继续。
- 抖一抖：向仍在托盘中的刚体施加短促冲量，不改变牌库。
- 撤回：将收纳槽最后一件玩具送回堆顶。

相机固定为俯视透视视角。竖屏时自动后退并提高，确保玩具箱和底部收纳槽同时可见。

每次点击开始或重新开始都会生成新的运行种子；牌库顺序、初始朝向和物理落点随之改变。关卡数量和每种玩具的三倍数约束保持不变，第一箱的两组教学目标仍保证可见，因此随机变化不会破坏可解性或首局教学。

## 8. UI 与视觉方向

视觉语言是“柔软塑料玩具 + 半透明收纳标签”：紫色玩具箱、糖果色玩具、圆角磨砂 HUD。3D 画布保留中央区域，HUD 只占顶部边缘、右侧工具轨和底部关键收纳槽。

收集与消除采用 3D/2D 混合表现：玩家先看到真实模型脱离物理堆并飞向对应屏幕槽位，抵达后才切换成稳定易读的 DOM 图标；第三件到达时三个图标向共同中心收拢、缩小，并配合彩色粒子爆开。这既保留“拿起实体”的空间感，也让三消结果在手机屏幕上足够清楚。

游戏根据浏览器语言自动选择中文或英文短文本。字体使用系统字体，避免 Poki 对外部字体请求的限制。

## 9. 音频

全部音效由 Web Audio API 即时合成，不下载音频文件：

- 选择：短促正弦音。
- 三消：随连击升高的三音和弦。
- 抖动、撤回、胜利和失败：独立合成反馈。

首次点击开始按钮时解锁 AudioContext，并提供静音按钮。

## 10. 技术架构

- `simulation/GameState.ts`：牌库、收纳槽、匹配、胜负与分数，完全不依赖 Three.js。
- `ToyboxGame.ts`：输入、场景适配、关卡流程和动画编排。
- `physics/PilePhysics.ts`：Rapier 世界、托盘边界、玩具碰撞体、固定步长同步和物理摇晃。
- `render/ToyFactory.ts`：GLB 加载、归一化、克隆和程序化模型。
- `render/TweenSystem.ts`：轻量、帧同步的补间动画。
- `ui/GameUI.ts`：DOM HUD、响应式布局、提示和结果界面。
- `platform/PokiBridge.ts`：可选 Poki 生命周期事件；本地无 SDK 时安全降级。

物件先按每局随机种子生成分层位置和朝向，再由 Rapier 重力、摩擦和碰撞自然落入托盘并形成遮挡关系。教学目标通过刚体重新放置到堆顶后继续参与重力和碰撞。游戏规则仍独立于物理世界；点击继续由 Three.js Raycaster 处理。移动端限制像素比，并允许刚体休眠以控制帧开销。

## 11. 资产与许可证

使用的社区资产均来自 Kenney，许可证为 CC0，可用于商业项目且无需署名：

- Cube Pets：猫、兔、熊猫奖励模型。
- Toy Car Kit：玩具赛车匹配模型。
- Brick Kit：低模拼搭积木匹配模型。

项目保留了每个资产包自带的许可证文件，位于 `public/licenses/`。运行时只包含 5 个精选 GLB，合计约 0.55 MB；不依赖外部 CDN。

## 12. Poki 接入计划

当前已实现安全的生命周期桥接：加载完成、玩法开始、暂停、胜负会在 Poki SDK 存在时发送事件。提交 Poki 后再把官方脚本加入定制 HTML，并通过 Poki Inspector 检查事件顺序。

正式测试前还应加入：

- `measure()`：关卡开始、首个三消、工具使用、失败和通关。
- 关卡间自然插屏机会。
- 失败后的可选激励广告，例如一次清空两个槽位；必须同时提供普通重开。
- 静音和输入在广告期间强制暂停。

## 13. 可玩原型验收标准

- npm 安装、开发运行和生产构建均成功。
- 桌面鼠标及移动触控均能选取和旋转。
- 三件相同玩具必定消除，七格满时失败，清空时胜利。
- 第一箱前两组三消有发光黑边目标引导，完成后恢复自由点击。
- 3D 玩具抵达槽位后才出现 2D 图标，三消聚拢和爆开反馈清晰完整。
- 每次开始或重开都会得到不同堆叠，同时保持牌库可三消。
- 抖动和撤回次数正确，失败后可立即重试。
- 三个关卡连续可玩，宠物模型正常登场。
- 模型加载失败时仍可使用程序化回退完成游戏。
- 竖屏和横屏均不遮挡关键玩法区域。
- 构建包不发起字体、模型或音频的第三方网络请求。
